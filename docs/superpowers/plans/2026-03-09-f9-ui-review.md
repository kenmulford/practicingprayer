# F-9 Comprehensive UI Review — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate inline style duplication across all views by introducing named styles, standardise typography/spacing, and fix two code bugs surfaced by the audit.

**Architecture:** All new visual tokens live in `Styles.xaml` as named `Style` resources. Views reference styles by key; no view should hard-code a colour, font size, or layout dimension that appears in more than one place. Bug fixes are isolated to the files that contain them.

**Tech Stack:** .NET MAUI 10, XAML resource dictionaries, C# code-behind.

**Branch:** `feature/f9-ui-review` → PR targets `dev` (integration branch; this project never merges feature branches directly to `master`).

> **BoxView note:** MAUI `BoxView` has two fill properties: `Color` (BoxView-specific) and `BackgroundColor` (inherited from `VisualElement`). Existing view files are inconsistent — some use `Color=`, some use `BackgroundColor=`. The `DividerLine` style uses `Color` (the dedicated property). When applying the style, remove whichever of `Color=` or `BackgroundColor=` is present inline, along with `HeightRequest=`, `Opacity=`, and `HorizontalOptions=` (all four are set by the style).

## Chunk 1 — Named Styles

### Task 1.1: Add button variant styles to Styles.xaml

**File:** `PrayerApp/Resources/Styles/Styles.xaml`

Insert the following block **immediately after** the closing `</Style>` of the existing `PrayerCardBorder` style (before the `<Style TargetType="Page" ...>` line).

> **Order matters:** `AnsweredButton` uses `BasedOn="{StaticResource SmallActionButton}"`. It MUST be declared after `SmallActionButton` in the dictionary. Do not reorder or split these blocks.

- [ ] **Add the 5 new Button styles in this exact order:**

```xml
<!-- ── Button variants ─────────────────────────────────────────────── -->

<!-- DangerButton: delete / destructive actions -->
<Style x:Key="DangerButton" TargetType="Button" BasedOn="{StaticResource {x:Type Button}}">
    <Setter Property="BackgroundColor" Value="IndianRed" />
    <Setter Property="TextColor" Value="White" />
</Style>

<!-- GhostButton: toolbar-style, transparent background -->
<Style x:Key="GhostButton" TargetType="Button" BasedOn="{StaticResource {x:Type Button}}">
    <Setter Property="BackgroundColor" Value="Transparent" />
    <Setter Property="TextColor" Value="{AppThemeBinding Light={StaticResource Primary}, Dark={StaticResource PrimaryDark}}" />
</Style>

<!-- SmallActionButton: compact in-card actions (Set Reminder, Edit) -->
<Style x:Key="SmallActionButton" TargetType="Button" BasedOn="{StaticResource {x:Type Button}}">
    <Setter Property="FontSize" Value="13" />
    <Setter Property="Padding" Value="8,6" />
</Style>

<!-- AnsweredButton: Mark Answered CTA — WarmGold + small.
     MUST follow SmallActionButton in this file (BasedOn dependency). -->
<Style x:Key="AnsweredButton" TargetType="Button" BasedOn="{StaticResource SmallActionButton}">
    <Setter Property="BackgroundColor" Value="{StaticResource WarmGold}" />
    <Setter Property="TextColor" Value="White" />
</Style>

<!-- CircleNavButton: Prayer Time previous/next arrows.
     Padding="0" overrides the default 14,10 — circle buttons need no internal padding.
     BackgroundColor and TextColor are left inline at each usage site (Prev vs. Next differ). -->
<Style x:Key="CircleNavButton" TargetType="Button" BasedOn="{StaticResource {x:Type Button}}">
    <Setter Property="WidthRequest" Value="60" />
    <Setter Property="HeightRequest" Value="60" />
    <Setter Property="CornerRadius" Value="30" />
    <Setter Property="Padding" Value="0" />
    <Setter Property="FontSize" Value="20" />
</Style>
```

<!-- ── Label variants ────────────────────────────────────────────────── -->

<!-- MutedText: secondary / de-emphasised text.
     Dark value uses Gray300 (#ACACAC) — lighter than Gray400 so it reads as
     "muted" on the dark OffBlack (#1f1f1f) background. -->

</Style>

    <Setter Property="HorizontalOptions" Value="Fill" />
</Style>

## Chunk 3 — Bug Fixes & Accessibility

### Task 3.1: Fix Settings.xaml.cs — missing await + redundant OnAppearing call

**File:** `PrayerApp/Views/Settings.xaml.cs`

Two bugs:

1. `btnClearSettings_Clicked` calls `DisplayAlertAsync(...)` without `await` — fire-and-forget.
2. Constructor manually calls `OnAppearing()` — MAUI calls it automatically; double-firing initialises the `Switch` twice before the page appears.

- [ ] **Replace the entire file with:**

```csharp
namespace PrayerApp.Views;

public partial class Settings : ContentPage
{
    public Settings()
    {
        InitializeComponent();
        // Removed: manual OnAppearing() call — MAUI invokes it automatically on page appearance
    }

    protected override void OnAppearing()
    {
        base.OnAppearing();
        chkSettingsAllowNotifications.IsToggled = PrayerApp.Services.Settings.AllowNotifications;
    }

    private async void btnClearSettings_Clicked(object sender, EventArgs e)
    {
        PrayerApp.Services.Settings.ClearSettings();
        await DisplayAlertAsync("Settings Cleared", "The next time the app runs all DB info will be reset.", "OK");
    }

    private void chkSettingsAllowNotifications_Toggled(object sender, ToggledEventArgs e)
    {
        PrayerApp.Services.Settings.AllowNotifications = e.Value;
    }
}
```
